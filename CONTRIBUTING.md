# Contributing

Thank you for considering a contribution! This project welcomes improvements to docs, UI polish, bug fixes, and small features.

## How to contribute
- Fork or create a feature branch from `main`.
- Make focused changes with clear commit messages.
- Ensure no secrets are committed. Use `.env.local` locally and `.env.example` as a reference.
- Open a Pull Request with a concise description and testing notes.

## Development
- Install: `npm install`
- Run dev server: `npm run dev`
- Build: `npm run build`
- Start: `npm start`

## Coding guidelines
- Keep changes minimal and well-scoped.
- Avoid breaking API routes; add new routes behind explicit paths.
- Include error handling and helpful logs where appropriate.

## Security
- Do not include credentials in commits or screenshots.
- If you find a vulnerability, please follow SECURITY.md.
